# 六面战役 · 像素美术实现规范 (ART_SPEC.md)

> 版本 v4.0 · 2026-05-13
> 适用范围：所有 UI 组件的代码绘制（Graphics API）与贴图制作
> 开发分辨率：720×1280 竖屏
> 渲染模式：NEAREST（贴图素材）/ 代码绘制直接在开发分辨率上操作

## 一、基础参数

### 1.1 双轨制说明

本项目采用 **代码绘制 + 贴图装饰** 混合方案：

| 方案 | 适用对象 | 坐标系 | 说明 |
|------|---------|--------|------|
| **代码绘制** | 面板/按钮/进度条/卡片 | 开发分辨率 720×1280 | Graphics API 直接画，数值=屏幕像素 |
| **贴图素材** | 纹理/图标/宝石/铆钉 | 32×32 原始 → ×2 缩放 = 64×64 屏幕 | NEAREST 渲染，保持像素锐利 |

### 1.2 核心参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 开发分辨率 | 720×1280 | 竖屏手游，所有代码绘制的坐标基准 |
| 贴图基础网格 | 32×32 px | 贴图素材的最小单元 |
| 贴图缩放倍率 | ×2 | 32→64px 屏幕尺寸 |
| 贴图渲染模式 | SCALE_MODES.NEAREST | 锐利无模糊 |
| 光照方向 | 左上 → 右下 | 高光在左上/上方，暗面在右下/下方 |
| 特效方式 | 像素粒子 | 不使用矢量发光，所有特效用像素粒子实现 |
| 像素单位 | 2px | 代码绘制中的最小可见单位（模拟像素感） |

### 1.3 像素单位换算

代码绘制时，为了保持像素风的"颗粒感"，定义 **1 像素单位 = 2px 屏幕像素**：

| 像素单位 | 屏幕像素 | 用途 |
|---------|---------|------|
| 1u | 2px | 最细描边、高光线、分隔线 |
| 2u | 4px | 标准描边、标准阴影 |
| 3u | 6px | 粗描边、强调边框 |
| 4u | 8px | 最粗边框（大面板外轮廓） |

后文所有数值均使用 **屏幕像素（px）** 标注，括号内标注像素单位（u）。

## 二、RPG 色卡系统

### 2.1 色卡来源与描边比例推导

色卡来源：xxNekoAnimexx 的 RPG Palettes（像素心形图标展示）

**描边比例推导**（从心形图标实测）：
- 心形图标原始尺寸：约 16×14 像素（含描边）
- 描边宽度：2px（占图标总宽的 12.5%）
- 填充区域：约 12×10 像素

**映射到我们的开发分辨率（720×1280）**：
- 32×32 贴图素材：描边 2px（原始像素），缩放×2后屏幕上 4px
- 代码绘制组件：描边 4px（2u），与贴图缩放后一致
- 描边比例统一为 **12.5%**（4px / 32px 组件最小单元）

### 2.2 色卡总览

每个调色板包含 5 色，从左到右排列。
注意：并非所有调色板都是同色系渐变，部分是主题搭配色组合。

## 三、描边与轮廓规范（开发分辨率 px）

### 3.1 描边粗细

| 组件类型 | 外轮廓 | 内边框 | 内线装饰 | 说明 |

| 中面板（卡片/技能框） | 2px (1u) 纯黑 | 4px (2u) 主题色（选中态） / 2px (1u) 暗色（未选中） | 外发光 4px | 中等 |
| 小面板（图标框/骰子槽） | 2px (1u) 纯黑 | 2px (1u) 深色 | 无 | 最薄 |
| 按钮 | 2px (1u) 纯黑 | 2px (1u) 暗色 | 无 | 紧凑 |
| 进度条 | 2px (1u) 纯黑 | 无 | 无 | 极简 |
| 分隔线 | 无 | 无 | 无 | 4px高（2px亮+2px暗） |

### 3.2 描边光照规则

所有描边遵循左上亮、右下暗的光照方向：
- 上边框：使用该色系的 Light 色
- 左边框：使用该色系的 Light 色
- 右边框：使用该色系的 Dark 色
- 下边框：使用该色系的 Darkest 色
- 左上角 2×2 区域：使用 Highlight 色（最亮点）
- 右下角 2×2 区域：使用 Darkest 色（最暗点）

## 七、排版规范（开发分辨率 px）

### 7.1 行距（Line Height）

| 文字层级 | 行距倍数 | 实际行距 |
|---------|---------|---------|
| H1 标题 | 1.3x | 42px |
| H2 副标题 | 1.3x | 32px |
| H3 小标题 | 1.4x | 28px |
| 正文 | 1.5x（最低） | 24px |
| 注释 | 1.4x | 20px |

硬规则：正文行距不得低于 1.5 倍字号。

### 7.2 内边距（Padding）— 文字距组件边缘

| 组件类型 | 上 | 右 | 下 | 左 | 最小值 |
|---------|---|---|---|---|--------|
| 大面板 | 24px | 32px | 24px | 32px | 24px |
| 中面板 | 16px | 24px | 16px | 24px | 16px |
| 小面板/图标框 | 12px | 16px | 12px | 16px | 12px |
| 按钮 | 16px | 32px | 16px | 32px | 16px |
| 卡片 | 20px | 24px | 20px | 24px | 20px |
| 弹窗 | 32px | 40px | 32px | 40px | 32px |

硬规则：文字距离组件内边缘最少 12px（6u）。

### 7.3 组件间距（Gap/Margin）

| 场景 | 间距 |
|------|------|
| 卡片之间 | 16-24px |
| 按钮之间 | 16px |
| 标题与内容 | 16-24px |
| 分隔线上下 | 16px |
| 图标与文字 | 16px |
| 屏幕边缘安全区 | 32-40px |

### 7.4 对齐规则

- 所有尺寸和位置必须是 2 的倍数（偶数），确保像素对齐
- 文字基线对齐
- 图标与文字垂直居中对齐时，图标中心与文字基线对齐

## 附录 B · 反模式清单

| 禁止 | 正确做法 |
|------|---------|
| 使用非偶数像素位置 | 所有坐标和尺寸为偶数 |

| 描边粗细不一致 | 同级组件描边粗细相同 |
| 文字无投影直接放在纹理上 | 所有文字必须带 2px 黑色投影 |
| 文字贴墙（padding < 12px） | 最小内边距 12px |
| 行距低于 1.5 倍 | 正文行距 >= 1.5x |
| 使用矢量发光/模糊特效 | 用像素粒子替代 |
| 颜色不在色卡范围内 | 所有颜色从色卡 5 级色阶中选取 |
| 高光和暗面方向不一致 | 统一左上亮、右下暗 |
| 投影方向随意 | 统一正 Y 轴（向下） |
| 代码绘制用32px网格数值 | 代码绘制直接用开发分辨率px值 |
| 描边1px在720p上看不见 | 最小可见单位2px（1u） |
